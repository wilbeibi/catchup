## Added
- Three new agents: **Cursor CLI**, **Cline**, and **Kimi Code**. Recap
  their sessions, search them, fork them natively, or hand them to any
  other agent — eight tools now work either end of a handoff
- `catchup fork kimi` from the wrong directory now tells you where the
  session lives and how to get there, instead of dying on kimi's own
  launch error (kimi is the one agent that binds sessions to the
  directory they started in)

## Fixed
- Recapping an OpenCode session while OpenCode was still running could
  miss the newest messages — or the whole conversation. Live sessions now
  read completely, tail included (the same fix ships in the new Cursor
  support, which stores history the same way)
- Kimi sessions recorded before kimi-code 0.26 had no working directory
  on file, so `--dir` and default selection couldn't find them; they now
  list and read like any other session
