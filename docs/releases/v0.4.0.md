## Added
- `fork` no longer needs a live session: seed a new agent from an old
  transcript file, something piped in, or a URL
- Point `fork` at a specific agent's directory, or pick from a list when
  more than one session could match

## Changed
- `catchup --help` and the skill are now grouped by what you're trying to
  do — recap a session, find the right one, or hand it off — instead of one
  flat list of flags

## Fixed
- Forking a huge transcript used to fail outright; now it's trimmed
  automatically so the fork still goes through
- Very large sessions could crash the fork instead of failing cleanly
- Fixed a leak where shared/presigned URLs could show up in output

**Full Changelog**: https://github.com/wilbeibi/catchup/compare/v0.3.0...v0.4.0
